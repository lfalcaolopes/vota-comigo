## Process

1. Read the test files below and AGENTS.md
2. Present a plan: which files to create/modify, in what order, and any architectural decisions
3. After I approve, implement the minimum code to make all tests pass
4. Run the tests and show me the final output

## Rules

- **Never modify test files.** If a test seems wrong, stop and ask me.
- **Never implement behavior no test requires.** No "while I'm here" additions.
- **Never skip running the tests.** Implementation is done only when all tests pass.
- Tests tell you **what** to do. AGENTS.md tells you **where** and **how** to structure it.
